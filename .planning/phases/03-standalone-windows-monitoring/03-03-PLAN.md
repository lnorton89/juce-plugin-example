<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView UI -> native source controller | User source requests cross from React into native message-thread handlers. |
| native persisted preferences -> startup restore | Local non-secret source preference influences startup behavior after enumeration. |
| native source diagnostics -> React UI | Bounded source state/error text crosses into UI display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | `source.select` payloads | mitigate | Parse protocol version, mode enum, bounded IDs, and request IDs before native action; reject unsupported VST3 source requests. |
| T-03-03-02 | Information Disclosure | source diagnostics and preferences | mitigate | Persist no secrets or raw hardware data beyond source ID/display metadata; bound error text and avoid raw unbounded COM/system strings. |
| T-03-03-03 | Denial of Service | UI status rendering | mitigate | Use bounded source lists/messages and a compact MUI strip; do not render one element per analyzer bin or unbounded diagnostics. |
| T-03-03-04 | Privacy | saved-source fallback | mitigate | Enforce D-14 and D-15: missing saved source starts stopped with choose-source state and never captures a different default device automatically. |
| T-03-03-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are planned; use existing React/MUI/Vitest dependencies. |
</threat_model>
