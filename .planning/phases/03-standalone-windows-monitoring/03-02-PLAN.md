<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Windows Core Audio COM -> native adapter | Endpoint IDs, device state, mix formats, packet flags, and HRESULT failures enter native code. |
| loopback capture thread -> analyzer ingress | Captured system audio crosses from a native capture thread into the analyzer path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Spoofing | endpoint identity matching | mitigate | Store and compare mode plus Core Audio endpoint ID; never match only display names. |
| T-03-02-02 | Information Disclosure | automatic endpoint fallback | mitigate | Enforce D-06 and D-15-compatible behavior: no switch to another render endpoint after selected-source failure. |
| T-03-02-03 | Denial of Service | mix-format conversion/capture packets | mitigate | Bound packet conversion buffers, reject unsupported formats visibly, and keep allocation/COM work outside the capture hot path. |
| T-03-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are planned; use Windows SDK Core Audio APIs already available to the native build. |
</threat_model>
