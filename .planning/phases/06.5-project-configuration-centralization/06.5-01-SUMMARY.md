# 06.5-01-SUMMARY: Configuration Schema, CMake Reader, and Validation Tooling

**Completed:** 2026-06-24

## What Was Built

### Central Project Config
- Added `project-config.json` at the repository root.
- Captures product identity, company/author metadata, version, plugin identifiers, runtime paths, URLs, UI text, bridge constants, Worker/D1 names, cloud resource prefix, token identifiers, build target names, dependency versions, and environment variable names.
- Preserves current project values so the first config slice changes the source of truth without changing product behavior.

### JSON Schema
- Added `project-config.schema.json` with `$version: "1.0.0"`.
- Documents required sections and field-level constraints for critical values such as semver, JUCE manufacturer/plugin codes, loopback dev-server URL, and environment variable naming.
- Records `filesAffected` metadata on leaf properties so future extraction work can trace each config value to its downstream consumers.

### CMake Integration
- Added `cmake/Config.cmake` with a small `read_config()` helper built on CMake's `string(JSON ...)` support.
- Root `CMakeLists.txt` now includes `cmake/Config.cmake` before `project()` and uses `${PROJECT_CONFIG_NAME}` and `${PROJECT_CONFIG_VERSION_STR}` for project identity.
- CMake validation fails fast with clear `FATAL_ERROR` messages for missing config fields, empty critical values, non-semver versions, invalid plugin code lengths, and schema version drift.

### Validation Tooling
- Added `scripts/validate-config.mjs`, a dependency-free Node validator for the project config/schema pair.
- Added `cmake/templates/ConfigValidation.cmake`, a standalone CMake script proving `Config.cmake` can load the root config outside a normal configure.
- The validator checks required fields, object shape, primitive types, min/max lengths, regex patterns, schema version, version string consistency, and `filesAffected` metadata on leaf schema properties.

## Deviations

- No external JSON Schema package was added. The validator implements the subset this project needs so a fresh checkout can validate configuration with only Node available.
- The schema traceability check enforces `filesAffected` on leaf config fields, where mappings are actionable. Container objects document grouping and rely on their children for concrete file mappings.
