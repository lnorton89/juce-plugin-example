<objective>
**Fix:** C++ source files, CMake build definitions, Worker code, and infrastructure configuration contain hardcoded product identity strings, namespace names, target names, and resource identifiers.

**What this plan delivers:** C++ sources consume config-driven constants via a generated header, PowerShell scripts consume a generated config module, Worker/infrastructure configs derive resource names from the centralized JSON. After this plan, changing values in `project-config.json` propagates to all C++ code paths, build targets, Worker code, and infrastructure manifest files.

**MVP vertical slice:** After this plan, a developer can change the product name in `project-config.json`, reconfigure CMake, and see the new name in the editor window title, bridge protocol, fallback error strings, and build target outputs — all without editing any source code directly.
</objective>
