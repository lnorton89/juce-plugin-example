    #ifndef LUMASCOPE_NATIVE_TESTS
    // Initialize licensing core
    if (auto keyRing = lumascope::PublicKeyRing::parse(LUMASCOPE_PUBLIC_KEY_RING_PATH))
    {

        auto store = std::make_unique<lumascope::LocalEntitlementStore>();

    #endif

        LUMASCOPE_PUBLIC_KEY_RING_PATH="${LUMASCOPE_PUBLIC_KEY_RING_PATH}")

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6-16 | Tampering | Cross-language fixture drift | mitigate | Automated test in both C++ and TypeScript verifies canonical JSON match. If either side changes, tests fail. CI should run both suites. |
| T-6-17 | Information Disclosure | Bridge protocol docs | mitigate | Documentation specifies bounded fields but does not include API keys, secrets, or production URLs. Example payloads use placeholder values. |
| T-6-18 | Denial of Service | Manual verification matrix | accept | Manual tests require user interaction but are documented step-by-step. Automated tests cover the same scenarios programmatically where possible. |
| T-6-19 | Elevation of Privilege | Both product targets | mitigate | Licensing code is shared between VST3 and standalone (same LicensingCore, ActivationClient). Both targets expose identical licensing capabilities through the same bridge protocol and UI components. Per D-08, licensing is always compact and analyzer-first in both targets. |
</threat_model>
