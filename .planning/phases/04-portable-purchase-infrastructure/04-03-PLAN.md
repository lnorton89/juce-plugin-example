Purpose: Ensure only trusted Lemon Squeezy entitlement events reach the D1 database and that retries, duplicates, and malicious payloads are handled safely.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request to Worker | Untrusted external POST requests arrive at /api/webhook/lemon-squeezy |
| Worker to D1 | Verified data flows from Worker logic to D1 database through typed repository functions |
| Worker external response | Error messages must not leak internal state, database contents, or configuration values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Spoofing | webhook.ts | mitigate | Verify Lemon Squeezy X-Signature with HMAC-SHA256 over exact raw request bytes using timing-safe comparison |
| T-04-12 | Tampering | webhook.ts | mitigate | Replay resistance: reject events with event_created_at older than 5 minutes from current time |
| T-04-13 | Tampering | webhook.ts | mitigate | Idempotency: check event_id in webhook_idempotency table before processing; return 200 for duplicates |
| T-04-14 | Denial of Service | webhook.ts | mitigate | Validate Content-Type is application/json; enforce maximum body size of 1 MB; reject oversized payloads with 413 |
| T-04-15 | Information Disclosure | webhook.ts / routes.ts | mitigate | Return generic error responses (400 Bad Request, 500 Internal Error) without details on failure; log specifics only to internal audit |
| T-04-16 | Tampering | repository.ts | mitigate | Use parameterized D1 prepared statements (env.DB.prepare().bind()) — never string-concatenate user data into SQL |
| T-04-17 | Spoofing | webhook.ts | mitigate | Validate event store_id, product_id, and variant_id against configured allowlists from manifest.yaml; reject unconfigured values |
| T-04-18 | Tampering | webhook.ts | mitigate | Use constant-time comparison (crypto.subtle.timingSafeEqual) for X-Signature verification; never use string equality |
| T-04-SC | Tampering | npm package installs | mitigate | No new npm packages for webhook logic — Web Crypto API is built into Workers runtime |

No high-severity threat may remain unmitigated when this plan completes.
</threat_model>

<task type="auto">
  <name>Task 2: Implement Lemon webhook signature verification and event handler</name>
  <files>worker/src/webhook.ts, worker/src/routes.ts, worker/src/index.ts</files>
  <read_first>worker/src/env.ts, worker/src/db/repository.ts, worker/src/routes.ts, worker/src/index.ts, AGENTS.md (Web Crypto guidance)</read_first>
  <action>Create worker/src/webhook.ts with the following exports:

   **verifyLemonSignature(request: Request, secret: string): Promise<{ valid: boolean; body: string }>**
   - Read the raw request body as text using request.text() (required for exact HMAC over raw bytes)
   - Get the X-Signature header from request.headers — normalize to lowercase for lookup
   - If X-Signature is missing, return { valid: false, body }
   - Compute HMAC-SHA256 using Web Crypto API:
     * const encoder = new TextEncoder()
     * const key = await crypto.subtle.importKey("raw", encoder.encode(secret), { name: "HMAC", hash: "SHA-256" }, false, ["verify"])
     * const signature = hexStringToArrayBuffer(xSignatureHeader)
     * const valid = await crypto.subtle.verify("HMAC", key, signature, encoder.encode(body))
   - Implement hexStringToArrayBuffer as a helper function
   - Do NOT implement timingSafeEqual manually — crypto.subtle.verify is constant-time
   - Return { valid, body }

   **parseLemonEvent(body: string): LemonWebhookEvent | null**
   - Parse body as JSON; if parsing fails, return null
   - Validate the Lemon Squeezy webhook event structure:
     * Must have meta.event_name (string), meta.custom_data (object), data (object)
     * data must have id, type, attributes
     * attributes must include store_id, first_subscription_item (for subscription events), or order (for order events)
   - Return a typed LemonWebhookEvent object with fields: eventName, eventId (from data.id), eventCreatedAt, storeId, productId, variantId, licenseKey (if present), status (mapped from Lemon event), customerName, customerEmail
   - Use try/catch for JSON.parse and return null on any field validation failure

   **handleLemonWebhook(request: Request, env: Env, ctx: ExecutionContext): Promise<Response>**
   - Content-Type check: return 400 if not application/json
   - Body size check: return 413 if Content-Length > 1,048,576 (1 MB)
   - Call verifyLemonSignature(request, env.LEMON_WEBHOOK_SECRET)
   - If invalid signature, write audit log (actor: "lemon_webhook", action: "invalid_signature") and return 401
   - Parse event with parseLemonEvent; if null, return 400
   - Replay check: check event_created_at — if older than 5 minutes from env time, reject with 400
   - Idempotency check: call repository.findIdempotency(eventId); if exists, return 200 (already processed)
   - Validate against configured Lemon identifiers: check store_id, product_id, variant_id against allowlists loaded from env vars or hardcoded defaults. If mismatched, write audit and return 200 with "skipped: unconfigured product" (not an error — webhook sender should not retry)
   - Process supported events: order_created, subscription_created, subscription_updated, subscription_cancelled, subscription_expired, license_key_created, license_key_updated
   - For each supported event, call repository.upsertFromWebhook() with mapped fields
   - Map Lemon status values: "active", "expired", "refunded", "disabled", "pending" map to LicenseStatus
   - Mark idempotency as completed via repository.markIdempotency()
   - Write audit log for the processed event
   - Return 200 { status: "processed", event: eventName, event_id: eventId }
   - For unsupported event types, mark idempotency as "skipped" and return 200

   Update worker/src/routes.ts:
   - Import handleLemonWebhook from webhook.ts
   - Wire the POST /api/webhook/lemon-squeezy route to call handleLemonWebhook
   - Wire the GET /api/health route to return { status: "ok", version: 1, environment: env.ENVIRONMENT }

   Update worker/src/index.ts:
   - Import routes and call handleRequest from the fetch handler
   - Implement CORS headers for OPTIONS requests and POST responses
   - Ensure all errors are caught and return 500 with generic { error: "internal_error" } envelope</action>
