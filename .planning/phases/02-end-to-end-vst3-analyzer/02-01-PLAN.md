<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host audio -> native analyzer | Untrusted host-provided block sizes, layouts, sample rates, and floating-point values enter native DSP. |
| analyzer config -> snapshot model | Future user-facing controls may alter config values; Phase 2 must keep fields bounded now. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `SpectrumAnalyzer::pushAudioBlock` and config construction | mitigate | Validate profile config ranges, clamp snapshot dB/value output, and test non-finite/silence/variable block inputs. |
| T-02-02 | Denial of Service | FFT/log-bin processing | mitigate | Use fixed FFT sizes and bounded snapshot bin capacity from `AnalyzerConfig`; no unbounded allocation in processing paths. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are planned; `juce_dsp` is from pinned JUCE. |
</threat_model>
