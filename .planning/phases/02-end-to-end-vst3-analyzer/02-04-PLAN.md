<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repository scripts -> local executable tools | pluginval path may be user-supplied and must not be treated as a secret or silently skipped. |
| VST3 host -> plug-in lifecycle | External host controls load, editor, resize, close/reopen, processing, and validation behavior. |
| manual smoke evidence -> project record | Human observations become release evidence and must distinguish pass, fail, skipped, and fallback. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Spoofing | `scripts/validate-plugin.ps1` pluginval path | mitigate | Print resolved executable path/version when possible and fail clearly on missing invalid paths. |
| T-02-10 | Repudiation | `02-HOST-SMOKE.md` | mitigate | Require explicit environment, tool, host, pass/fail, limitation, and timestamp fields before approval. |
| T-02-11 | Denial of Service | Host lifecycle | mitigate | Manual and automated lifecycle checks cover editor close/reopen/resize/destroy while audio continues. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are planned; pluginval is an external executable path, not a package-manager install. |
</threat_model>
